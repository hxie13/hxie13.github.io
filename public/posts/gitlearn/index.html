熟悉git命令

Git commit

git 提交文件时追求轻量化，在条件允许情况下，它将当前版本与仓库中的上一版本进行对比，并把所有差异打包一起作为一个提交记录(一次git commit)。在项目开发过程中，Git会保存历史提交记录，作为项目不同阶段的快照，我们可以在这些提交记录上快速切换。

分支创建及跳转在新分支开发

  • git branch newOne + git checkout newOne
  • git checkout -b newOne

此外， git checkout也可以被git switch替代实现

分支合并

在不同分支开发过程中，不同分支上拥有不同的功能实现，为将所有实现功能合并，需要将分支进行合并.

  • git merge;
  • git rebase(创建线性记录提交表);

分离Head

  • HEAD默认指向分支名所在的最近一次提交记录上，但可以改变其位置，依旧是通过git checkout命令，指定提交记录的哈希值来实现跳转。
  • 查看提交记录的哈希值：git log，每次改变HEAD指向都需要通过该方式查询提交记录的哈希值十分麻烦，虽然提交时只需要提供记录的前几个字符即可。

相对引用^

在提交记录后加上^即可返回其上一次提交记录上

git fetch

  • 从远程仓库中下载本地仓库中缺失的提交记录
  • 更新远程分支指针

但执行这一操作仅会下载远程数据，并不会修改本地文件，即本地仓库并未与远程仓库同步，需要额外操作完成这一任务。
